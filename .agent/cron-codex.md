# 📜 DVIEW Cron Operational Codex (디뷰 크론봇 수행 지침)

이 문서는 DVIEW 프로젝트의 5분 주기 크론봇(AI 에이전트)이 깨어날 때마다 읽고 수행해야 하는 **최상위 마스터피스(S+ Grade) 자동화 개선 가이드**입니다. 디뷰 서비스의 핵심 기능(실거래가 분석, 갭투자 탐색, 학군/인프라 큐레이션, 바이럴 페이월, B2B 제휴 광고)에 맞추어 재정의되었습니다.

---

## 1. 🔍 크론 자동화 개선 루프의 5대 핵심 감사 영역

에이전트가 깨어났을 때 수행하는 분석 및 수정 프로토콜은 다음과 같이 다중 레이어에서 작동합니다.

### ① 데이터 정합성 & 무결성 방어막 (Data Integrity & Robustness)
* **실거래가 & 거래량 데이터 예외 처리**: 외부 API나 정적 데이터에서 비정상적인 거래 금액(예: `0`, `null`, `NaN`) 또는 취소 거래건 유입 시 `TransactionTable.tsx`나 `TransactionChartSection.tsx`에서 차트 및 테이블 렌더링 크래시가 발생하지 않는지 방어 로직 검사.
* **갭투자 & 전세가율 연산 방어**: 매매가와 전세가 차이를 계산하는 갭투자 탐색기(`GapInvestmentExplorer.tsx`) 및 실거래 통계 연산 시 분모가 0이 되어 발생할 수 있는 Zero-Division 오류 및 NaN 데이터 처리 확인.
* **좌표 기반 거리 연산 안전성**: 유치원/어린이집 위치 정보를 표시하는 `ChildcareDetailSection.tsx` 등에서 `haversineDistance` 및 `parseCoordString` 함수가 손상된 좌표값이나 빈 문자열 유입에 대해 `NaN`을 반환하지 않도록 예외 처리 보완.

### ② 성능 및 지연율 최적화 (Zero-Jank Execution)
* **대용량 실거래 목록 리렌더링 최적화**: 대량의 실거래 데이터(예: 롯데캐슬 184건 등)가 로드될 때, 모바일 뷰포트에서 스크롤 성능이 저하되지 않도록 `react-window` 가상화 테이블(`TransactionTable.tsx`)의 행 렌더러 메모이제이션(`useMemo`, `React.memo`) 최적화 상태 유지.
* **SWR 데이터 동기화 루프 감시**: 잦은 단지 전환이나 실시간 라운지 피드(`LoungeFeedClient.tsx`) 갱신 시 중복 네트워크 패칭과 이로 인한 메인 스레드 블로킹(16.7ms 초과) 현상을 모니터링하여 불필요한 API 요청 제어.

### ③ 어반 에메랄드 & 파스텔 큐트 (Urban Emerald & Pastel Cute) 디자인 & UX 고도화
* **브랜드 비주얼 토큰 및 명도비 준수**: DVIEW 시그니처 색상인 어반 에메랄드(`#00a37b`) 및 파스텔 큐트 계열 색상이 하드코딩되지 않고 CSS 변수와 Tailwind 토큰으로 동기화되어 있는지 검사. 텍스트 가독성을 위한 웹 접근성 대비율(WCAG AA 4.5:1 이상) 준수 여부 정밀 진단.
* **모바일 텍스트 래핑 및 가로 오버플로우 방지**: 모바일 뷰포트 너비(360px ~ 430px) 내에서 아파트 상세 정보, 실거래가 금액, 갭투자 매칭률 텍스트(예: "매칭률 60% 매칭") 등이 영역 밖으로 깨져 흐르거나 불필요한 가로 스크롤을 유발하지 않도록 자동 텍스트 줄바꿈(Text Wrapping) 및 반응형 레이아웃 보완.

### ④ 바이럴 공유 및 광고 수익화 무결성 (Growth Engine & Monetization)
* **1-Click 공유 및 이미지 캡처 검증**: 카카오톡 단톡방용 텍스트/링크 요약 복사 기능 및 단지 요약 카드 이미지 캡처(`html2canvas-pro` 연동) 기능이 모바일 브라우저 환경에서 중단 없이 원활하게 가동되는지 검사하고, 바이럴 게이트(`ViralPaywallGate.tsx`) 동작 상태를 점검.
* **B2B 제휴 배너 및 애드센스 CLS 제어**: `ContextualB2BAdBanner.tsx` 및 광고 슬롯 로딩 지연으로 인해 갑작스러운 레이아웃 이동(Cumulative Layout Shift)이 발생하여 유저 경험을 파괴하지 않도록 고정 뷰포트/플레이스홀더 영역 보존 여부 검사.

### ⑤ 라운지 커뮤니티 활성화 및 UGC 엔진 (Lounge Community Engagement)
* **글쓰기 및 댓글 피드백 반응성**: 유저가 거주 후기, 단지 소식을 작성하는 `LoungeComposeClient.tsx` 및 댓글 섹션(`CommentSection.tsx`)에서 작성 버튼 클릭 시 지연 현상이 없도록 반응성을 보장하고, 피드백 완료 후 화면 갱신 최적화.
* **커뮤니티 내비게이션 최적화**: 단지별 라운지 피드(`LoungeFeedClient.tsx`)와 상세 페이지(`LoungeDetailClient.tsx`)의 로딩 속도를 분석하여 불필요한 레이아웃 깜빡임과 빈 화면 노출 시간 최소화.

## 2. 🛡️ 예외 발생 시의 자가 복구(Rollback) 안전장치

크론봇이 코드를 스스로 수정하다가 시스템을 망가뜨리는 것을 방지하기 위해, 무조건 다음의 **안전 하네스(Safety Harness)**를 통과한 코드만 반영합니다.

1. **수정 후 빌드 검증**: `tsc --noEmit` 실행 결과 에러가 단 1개라도 발생하면 즉시 변경 사항 롤백 (`git reset --hard`).
2. **린트 검증**: `npm run lint` 통과 실패 시 즉시 롤백.
3. **테스트 슈트 검증**: Jest 유닛 테스트 및 Playwright E2E 테스트(`npm run test:e2e`) 중 실패하는 케이스가 생기면 즉시 롤백.
4. **결과 반영**: 모든 테스트가 완벽히 초록불(Green)인 경우에만 `git commit` 실행 (단, 원격지 푸시는 사용자의 명시적인 '푸시' 지시 규정을 우선적으로 준수함).
5. **자가 예약 연속 실행 (Self-scheduling Chain)**: 위 모든 단계와 커밋 처리가 완료되면, 에이전트는 즉시 5초 타이머(`schedule` 툴)를 예약하여 꼬리를 무는 형태로 다음 자가 개선 루프를 딜레이 없이 연속 가동시킵니다.
