# Plan - Page Transition and ApartmentModal Optimization

## Objective
Enhance page-to-page transitions and optimize the rendering speed/responsiveness of `ApartmentModal` in the D-VIEW web application, ensuring 100% build and E2E test integrity.

## Milestones

### Milestone 1: Codebase Exploration and Diagnostics
- **Tasks**:
  - Search for `ApartmentModal` and its usage in the codebase.
  - Locate page routing, `Link` components, service worker `public/sw.js`, and data fetching/caching logic (SWR, Context).
  - Profile existing bottlenecks and design the specific solution architecture.
- **Verification**: Exploration report generated by Explorer subagent.

### Milestone 2: Page Transition Optimization (R1)
- **Tasks**:
  - Optimize Next.js Link prefetching policy.
  - Implement programmatic prefetching on hover of navigation links or target routes.
  - Update `public/sw.js` service worker caching strategy to effectively cache static JS chunks and JSON data.
  - Resolve duplicate fetches/state resets on transitions by auditing SWR/Context caching.
- **Verification**: Worker implementation handoff with verification results.

### Milestone 3: ApartmentModal Optimization (R2)
- **Tasks**:
  - Split `ApartmentModal` and heavy components (e.g., Recharts, comments, upload, calculators) using dynamic imports (`next/dynamic`).
  - Implement a preload mechanism that triggers chunk and data fetching when a user hovers or focuses on an apartment item *before* clicking.
  - Apply `React.memo`, `useMemo`, and `useCallback` on heavy components (charts, tables, list items) to prevent redundant rendering and scrolling lag.
- **Verification**: Worker implementation handoff, Reviewer verification, and Challenger correctness checks.

### Milestone 4: Build & Test Validation (R3)
- **Tasks**:
  - Run the production build command (`npm run build`).
  - Run E2E tests (`performance-ux.spec.ts` and `routing-bug.spec.ts`).
  - Perform integrity audit with Forensic Auditor.
- **Verification**: Complete passes of E2E tests and successful Next.js production build.
